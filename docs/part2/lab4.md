**50분 타이머.** 오늘 처음으로 **흐름을 처음부터 짓습니다.**

---

## 준비

테스트용 **신규 지원자 이력서 PDF**(강사 제공, 시드 42명과 다른 인물)를 내려받아 둡니다.

---

## 단계

1. 브라우저에서 Outlook(`https://outlook.office.com`)을 엽니다.

    ![촬영: Outlook 메일 화면](https://placehold.co/736x490/EEE/31343C "lab4-01")

2. 왼쪽 폴더 영역에서 **새 폴더**를 만들고 이름을 `채용지원[샘플]`로 지정합니다.

    ![촬영: 새 폴더 '채용지원[샘플]' 생성](https://placehold.co/736x490/EEE/31343C "lab4-02")

3. 오른쪽 위 **설정(톱니)** → **메일** → **규칙** → **+ 새 규칙 추가**를 클릭합니다.

    ![촬영: 메일 규칙 추가 화면](https://placehold.co/736x490/EEE/31343C "lab4-03")

4. 규칙을 아래처럼 설정하고 **저장**합니다. (`[이력서샘플]` 제목 메일이 자동으로 폴더로 들어가 흐름이 발화합니다.)

    - **이름**: `이력서 샘플 분리 규칙`
    - **조건**: 제목에 다음 포함 → `[이력서샘플]`
    - **동작**: 다음 폴더로 이동 → `채용지원[샘플]`

    ![촬영: 규칙 설정](https://placehold.co/736x490/EEE/31343C "lab4-04")

    {: .note }
    "이 메일이 지원 메일인가?"를 흐름이 판단하지 않습니다. **분류는 Outlook 규칙(상류)**이 하고, 폴더에 들어온 것은 이미 지원 메일이라고 신뢰합니다 — 그래서 흐름이 단순해집니다.

5. Copilot Studio 왼쪽 메뉴 **흐름**에서 **+ 새 에이전트 흐름**을 클릭합니다. (이름은 저장할 때 붙입니다.)

    ![촬영: 새 에이전트 흐름 만들기](https://placehold.co/736x490/EEE/31343C "lab4-05")

6. 트리거 검색창에 `새 메일이`를 입력하고 Office 365 Outlook **새 메일이 도착하면 (V3)**을 선택합니다.

    ![촬영: '새 메일이 도착하면 (V3)' 선택](https://placehold.co/736x490/EEE/31343C "lab4-06")

7. **연결**을 본인 계정(본인 메일)으로 연결합니다.

    ![촬영: Outlook 연결(본인 계정)](https://placehold.co/736x490/EEE/31343C "lab4-07")

8. 트리거 고급 매개변수를 펼쳐 설정합니다. **폴더** = `채용지원[샘플]`, **첨부 파일 포함** = 예, **첨부 파일이 있는 항목만** = 예.

    ![촬영: 트리거 옵션(폴더·첨부 예)](https://placehold.co/736x490/EEE/31343C "lab4-08")

    {: .important }
    **폴더 = 채용 전용 폴더**여야 일반 업무 메일이 흐름을 깨우지 않습니다. **첨부 파일 포함 = 예**여야 다음 단계에서 PDF 내용을 읽습니다.

9. 오른쪽 위 **저장**을 클릭하고, 이름 입력 창에 `적재 흐름`을 입력합니다.

    ![촬영: 흐름 이름 = 적재 흐름](https://placehold.co/736x490/EEE/31343C "lab4-09")

10. 트리거 아래 **+ 작업 추가** → AI Builder **프롬프트 실행**을 추가합니다.

    ![촬영: AI Builder '프롬프트 실행' 추가](https://placehold.co/736x490/EEE/31343C "lab4-10")

11. AI Builder **연결**을 본인 계정으로 연결합니다.

    ![촬영: AI Builder 연결](https://placehold.co/736x490/EEE/31343C "lab4-11")

12. **프롬프트** 드롭다운에서 제공된 **이력서 문서를 읽고 요약하는 프롬프트**를 선택합니다. (이 프롬프트의 내용은 아래 참조)

    ![촬영: 이력서 요약 프롬프트 선택](https://placehold.co/736x490/EEE/31343C "lab4-12")

    {: .note }
    **제공 프롬프트 내용 (참조)** — 비정형 이력서 → 정형 필드로 변환하는 추출기입니다.
    ```
    당신은 이력서 분석 전문가입니다.
    첨부된 이력서 문서를 분석하여 아래 JSON 형식으로만 출력하세요.

    ### 추출 규칙:
    - 지원자명: 이력서에서 이름 추출
    - 지원직군: [영업/마케팅, 기획/전략, 인사/HR, 재무/회계, IT/개발, 물류/SCM, 고객서비스, 기타] 중 하나
    - 경력사항: [신입, 1~3년, 3~5년, 5년 이상] 중 하나
    - 이력서요약: 핵심 경력·강점을 200자 내외로. 정량 수치(%, 금액, 건수)는 일반화 말고 그대로 보존
    ### 출력 형식:
    { "지원자명":"", "지원직군":"", "경력사항":"", "이력서요약":"" }
    ```
    경력사항·지원직군이 **고정된 선택지로만 출력**되므로, 뒤에서 선택형 컬럼(CareerLevel)에 그대로 매핑해도 값이 일치합니다. 모델은 **GPT-4.1 mini**.

13. 프롬프트 입력 **Resume** 칸에 `/`를 입력하고 트리거의 **첨부 파일 콘텐츠**(배열)를 넣습니다.

    ![촬영: 프롬프트 Resume = 첨부 파일 콘텐츠(배열)](https://placehold.co/736x490/EEE/31343C "lab4-13")

    {: .important }
    배열을 입력에 연결하면 **CS가 자동으로 반복(Apply to each)을 생성**합니다. 반복을 따로 먼저 만들 필요가 없습니다 — 아래 파일 만들기·항목 만들기는 이 **자동 생성된 반복 안에서** 추가합니다.

14. 자동 생성된 **반복 안에서** **+ 작업 추가** → SharePoint **파일 만들기**를 추가하고 본인 계정으로 연결합니다.

    ![촬영: SharePoint 파일 만들기 추가](https://placehold.co/736x490/EEE/31343C "lab4-14")

15. **사이트 주소** 칸에서 **동적 콘텐츠 → 환경 변수**의 **`SPSiteUrl`** 칩을 선택합니다. (URL 직접 입력 대신 환경 변수)

    ![촬영: 사이트 주소 = SPSiteUrl 환경 변수](https://placehold.co/736x490/EEE/31343C "lab4-15")

16. **폴더 경로**에서 `이력서 보관함`을 선택합니다.

    ![촬영: 폴더 경로 = 이력서 보관함](https://placehold.co/736x490/EEE/31343C "lab4-16")

17. **파일 이름** 칸의 **fx**를 열고 아래 식을 붙여넣은 뒤 **초록 체크**가 뜨면 **삽입**합니다.

    ```
    concat(formatDateTime(utcNow(),'yyyyMMdd_HHmmss'), '_', substring(guid(),0,8), slice(items('반복_적용')?['name'], lastIndexOf(items('반복_적용')?['name'], '.')))
    ```

    ![촬영: 파일 이름 수식 + 초록 체크](https://placehold.co/736x490/EEE/31343C "lab4-17")

    {: .important }
    **원본 파일명을 쓰지 않습니다.** 공백이 있으면 링크가 잘려 깨집니다(실제 겪은 버그). `{수신시각}_{GUID8}{확장자}`로 공백 없는 고유명을 만듭니다. 확장자는 2-인수 substring이 안 돼 `slice(...)`로 뽑습니다.

18. **파일 콘텐츠** 칸에 `/`를 입력하고 트리거의 **첨부 파일 콘텐츠**를 넣습니다. (동적 콘텐츠에서 트리거 항목 **더 보기**를 눌러야 보입니다.)

    ![촬영: 파일 콘텐츠 = 첨부 파일 콘텐츠](https://placehold.co/736x490/EEE/31343C "lab4-18")

19. 반복 안에 **+ 작업 추가** → SharePoint **항목 만들기**를 추가합니다.

    ![촬영: SharePoint 항목 만들기 추가](https://placehold.co/736x490/EEE/31343C "lab4-19")

20. **사이트 주소** = **`SPSiteUrl`** 환경 변수 칩, **목록 이름** = **본인 지원자 목록**을 선택합니다.

    ![촬영: 사이트(SPSiteUrl)·본인 목록 지정](https://placehold.co/736x490/EEE/31343C "lab4-20")

21. **제목(Title)** 칸의 **fx**를 열고 아래를 입력합니다. `지원자명`·`지원직군`은 식 안에서 **동적 콘텐츠 칩**으로 삽입합니다(키보드 텍스트 아님).

    ```
    concat('[', formatDateTime(utcNow(),'yyyy-MM-dd'), ']', «지원자명», '_', «지원직군»)
    ```

    ![촬영: 제목 수식(칩 삽입)](https://placehold.co/736x490/EEE/31343C "lab4-21")

22. **ApplicantName** 칸에 `/` → `지원자명` 칩을 넣습니다.

    ![촬영: ApplicantName = 지원자명](https://placehold.co/736x490/EEE/31343C "lab4-22")

23. **Email** 칸에 `/` → `이메일` 칩을 넣습니다.

    ![촬영: Email = 이메일](https://placehold.co/736x490/EEE/31343C "lab4-23")

    {: .warning }
    Email 칸에 **엉뚱한 칩(예: "시작")이 들어가기 쉽습니다.** 반드시 AI 출력의 **이메일**인지 확인하세요.

24. **JobPosition** 칸에 `/` → `지원직군` 칩을 넣습니다.

    ![촬영: JobPosition = 지원직군](https://placehold.co/736x490/EEE/31343C "lab4-24")

25. **CareerLevel** 칸에 `/` → `경력사항` 칩을 넣습니다. (프롬프트가 신입/1~3년/3~5년/5년이상 중 하나로 고정 출력 → 선택 컬럼과 일치)

    ![촬영: CareerLevel = 경력사항](https://placehold.co/736x490/EEE/31343C "lab4-25")

26. **AIResumeSummary** 칸에 `/` → `이력서요약` 칩을 넣습니다.

    ![촬영: AIResumeSummary = 이력서요약](https://placehold.co/736x490/EEE/31343C "lab4-26")

    {: .warning }
    `AIResumeSummary`는 **일반 텍스트** 컬럼이어야 합니다(Lab 1 환경에서 설정). 서식 있는 텍스트면 요약이 `<div class="ExternalClass…">` 태그와 함께 저장돼 승인 카드·챗봇에서 태그가 노출됩니다.

27. **ResumeLink** 칸의 **fx**를 열고 아래 식을 입력합니다. (환경 변수 사이트 URL + 파일 만들기가 반환한 경로)

    ```
    concat(parameters('SPSiteUrl'), outputs('파일_만들기')?['body/Path'])
    ```

    ![촬영: ResumeLink 수식](https://placehold.co/736x490/EEE/31343C "lab4-27")

    {: .note }
    **ReviewStatus(전형단계)·AIFitLevel·OnHold(보류)는 매핑하지 않습니다** — 각각 컬럼 기본값(검토중 / 미적용 / false)으로 시작하고, 이후 단계가 채웁니다.

28. 오른쪽 위 **저장**을 클릭합니다.

    ![촬영: 흐름 저장](https://placehold.co/736x490/EEE/31343C "lab4-28")

29. **흐름 검사기**로 오류·경고가 없는지 확인합니다.

    ![촬영: 흐름 검사기](https://placehold.co/736x490/EEE/31343C "lab4-29")

30. **게시**를 클릭합니다.

    ![촬영: 게시 완료](https://placehold.co/736x490/EEE/31343C "lab4-30")

31. 준비한 **이력서 PDF**를 첨부하고 제목을 `[이력서샘플] 홍길동`처럼 해서 **내 메일로 보냅니다.**

    ![촬영: [이력서샘플] 제목 + PDF 첨부 메일 발송](https://placehold.co/736x490/EEE/31343C "lab4-31")

    {: .note }
    규칙이 안 옮기면 받은 편지함에서 그 메일을 `채용지원[샘플]` 폴더로 직접 끌어다 넣으세요(트리거 발화).

32. 흐름의 **실행 기록**에서 성공으로 돌았는지 확인합니다.

    ![촬영: 흐름 실행 성공 기록](https://placehold.co/736x490/EEE/31343C "lab4-32")

33. **본인 지원자 목록**을 열어 **새 항목**이 생겼는지 확인합니다(요약·링크 채워짐, 상태 **보류 중**).

    ![촬영: 본인 목록 새 항목(보류 중)](https://placehold.co/736x490/EEE/31343C "lab4-33")

## 확인

- [ ] 흐름이 게시됐다(트리거 → AI 프롬프트(자동 반복) → 파일 만들기 → 항목 만들기)
