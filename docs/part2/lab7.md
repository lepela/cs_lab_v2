# Lab 7. 면접 확정 — 토픽 (대화형 HITL) 💬

> **이번 랩 완성물**: 챗봇 대화로 게이트(동명이인·승인됨)를 거쳐 **확인 카드**로 전형단계를 바꾸는 토픽
> **예상 시간**: 60분 · **완성 신호**: "○○○ 서류합격 처리해줘" → 확인 카드 → 전형단계 변경 + 승인 보존

**60분 타이머.** Lab 6에서 만든 흐름에 **게이트와 확인 카드**를 얹어 면접 확정을 완성합니다.

---

## 준비

Lab 6에서 만든 **전형단계 변경 흐름**(도구로 붙은 상태)이 필요합니다. 흐름은 "ID로 바꾸고 승인 보존"만 합니다 — 여기서 그 앞단(누구인지·자격·확인)을 토픽으로 짓습니다.

{: .note }
**3층 분담** — 오케스트레이터(조회·동명이인 되묻기·미적용 평가) / **토픽**(게이트·확인 카드) / 흐름(ID 단건 갱신). 불변식은 확률적 LLM이 아니라 토픽·흐름이 **결정적으로** 지킵니다.

---

## 오케스트레이터 지침

1. 면접관 에이전트 **지침**에 아래 `## 전형단계 변경` 섹션을 이어 붙이고 **저장·게시**합니다.

    ```
    ## 전형단계 변경
    - 전형단계 변경 요청 시: 지원자를 조회해 동명이인이면 누구인지 되묻고, AI적합도가 미적용이면 이력서요약을 루브릭에 대조해 등급을 먼저 정한 뒤 '전형단계 변경 확인' 토픽을 호출한다(AI적합도 함께 전달).
    - AI적합도는 사람에게 묻지 않는다. 사용자가 적합도 직접 지정·미적용 변경을 요구하면 거절하고 "평가를 수행하면 자동 갱신됩니다"로 안내한다.
    - 여러 명·전원 일괄 변경은 거절하고 한 명씩 지정하게 한다.
    ```

    ![촬영: 오케스트레이터 지침 전형단계 변경 섹션](https://placehold.co/736x490/EEE/31343C "lab7-01")

    {: .note }
    **dedup 다중방어** — 동명이인 되묻기는 여기(오케스트레이터=front), 토픽의 count=1 게이트(=back)가 한 번 더. 웹의 front/back validation처럼 두 겹입니다.

---

## 토픽 골격 붙이고 노드 하나씩 검증

2. 새 토픽 **`전형단계 변경 확인`** 을 만들고, **코드 보기**에서 제공된 **검증 골격 YAML**(전형단계 5값 버전)을 붙여넣습니다.

    ![촬영: 토픽 코드 보기에 골격 YAML 붙여넣기](https://placehold.co/736x490/EEE/31343C "lab7-02")

    {: .note }
    이 골격은 검증된 것입니다(write 회귀·무한루프 등 함정 해소). 붙여넣은 뒤 아래 노드를 **하나씩 열어 확인·구성**합니다. 토픽 구조:
    ```
    Question(이름) → 조회 → 게이트①(count=1) → 게이트②(승인됨)
      → Question(전형단계) → 적응형 카드(확정/취소)
      → 확정: 흐름 호출(ID·전형단계·AI적합도) + 모든 토픽 종료 / 취소: 종료
    ```

3. **Question(지원자이름)** — 대상 지원자 이름을 받는지 확인합니다.

    ![촬영: Question 지원자이름](https://placehold.co/736x490/EEE/31343C "lab7-03")

4. **항목 가져오기(조회)** — 본인 계정 **연결**, 사이트=SPSiteUrl, 목록=본인 목록, 필터 `ApplicantName eq '{지원자이름}'` 을 확인합니다.

    ![촬영: 조회 노드 연결·필터](https://placehold.co/736x490/EEE/31343C "lab7-04")

5. **게이트 ① (행 갯수)** — 조회 결과 count로 분기: **0건 → "없음"**, **2건↑ → "동명이인, 이메일/ID로 특정"**, **1건만 통과**.

    ![촬영: 게이트① count 분기](https://placehold.co/736x490/EEE/31343C "lab7-05")

6. **게이트 ② (승인됨 전제)** — 승인상태 ≠ 승인됨이면 "AI 요약 승인이 완료되지 않은 지원자"로 차단합니다.

    ![촬영: 게이트② 승인됨 전제](https://placehold.co/736x490/EEE/31343C "lab7-06")

7. **Question(전형단계)** — 닫힌목록 **5값(검토중/서류합격/면접합격/최종합격/불합격)** 인지 확인합니다.

    ![촬영: Question 전형단계 닫힌목록](https://placehold.co/736x490/EEE/31343C "lab7-07")

8. **적응형 카드** — **지원자이름·현재 전형단계·AI적합도**가 변수로 바인딩되고 **[확정]·[취소]** 버튼이 있는지 확인합니다.

    ![촬영: 적응형 카드 바인딩](https://placehold.co/736x490/EEE/31343C "lab7-08")

    {: .note }
    한 카드가 지원자마다 다른 내용으로 그려집니다. "어떤 상태로 바꿀지"도 카드/질문이 담당하므로 오케스트레이터가 전형단계를 미리 묻지 않습니다.

9. **확정/취소 분기** — **확정 → 흐름 호출** / **취소 → "변경을 취소했습니다"** 인지 확인합니다.

    ![촬영: 확정/취소 분기](https://placehold.co/736x490/EEE/31343C "lab7-09")

10. **흐름 호출** — Lab 6의 **전형단계 변경 흐름**이 연결됐는지, 입력에 **ID·전형단계·AI적합도**가 전달되는지 확인합니다(흐름 ID 재연결이 필요할 수 있음).

    ![촬영: 흐름 호출 입력 매핑](https://placehold.co/736x490/EEE/31343C "lab7-10")

11. 확정·취소 분기 끝에 **모든 토픽 종료** 노드가 있는지 확인하고 **저장·게시**합니다.

    ![촬영: 모든 토픽 종료 + 저장·게시](https://placehold.co/736x490/EEE/31343C "lab7-11")

    {: .warning }
    **모든 토픽 종료**가 없으면 토픽이 자신을 다시 부르는 **재귀 호출**이 생깁니다. 반드시 확인하세요.

## 테스트 — 4가지 시나리오

12. **정상(미적용 포함)** — 승인됐지만 AI적합도가 **미적용**인 지원자에게 `○○○ 서류합격 처리해줘`. ① 조회·서술 ② 미적용 감지 ③ 루브릭 자동 평가 ④ 카드 ⑤ 확정 → **전형단계·AI적합도 동시 갱신 + 승인 보존**되는지 확인.

    ![촬영: 정상 경로](https://placehold.co/736x490/EEE/31343C "lab7-12")

13. **없는 이름(0건)** — 존재하지 않는 이름 → 게이트①에서 차단(갱신 X).

    ![촬영: 0건 차단](https://placehold.co/736x490/EEE/31343C "lab7-13")

14. **미승인 차단** — 아직 승인 안 된 지원자 → 게이트②에서 차단.

    ![촬영: 미승인 차단](https://placehold.co/736x490/EEE/31343C "lab7-14")

15. **일괄 거절 + 승인 보존** — `전원 불합격으로 바꿔줘` → 오케스트레이터가 거절. 정상 변경 후 그 지원자의 승인상태가 **보존**됐는지(Lab 6 흐름 덕분에 리셋이 안 일어남) 확인.

    ![촬영: 일괄 거절 / 승인 보존](https://placehold.co/736x490/EEE/31343C "lab7-15")

## 확인

- [ ] 토픽 게이트(count=1·승인됨)가 동명이인·없음·미승인을 **결정적으로** 차단한다
- [ ] 적응형 카드 확정 시 전형단계·AI적합도가 갱신되고, 취소 시 아무것도 안 바뀐다
- [ ] 미적용 지원자는 오케스트레이터가 자동 평가 후 카드로 넘긴다
- [ ] 일괄 변경은 거절되고, 모든 토픽 종료로 재귀가 차단된다

**읽기는 커넥터+지침으로, 상태를 바꾸는 일은 흐름으로.** 같은 "사람 끼워 넣기"도 비동기(Lab 5)와 대화형(여기)으로, 같은 확인도 텍스트와 카드로 갈립니다. 그 경계를 손으로 그어 본 것 — 이 과정이 남기는 핵심입니다. (회고는 피날레에서)
