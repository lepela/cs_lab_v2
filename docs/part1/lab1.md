# Lab 1. 환경 받기 📦

> **이번 랩 완성물**: 내 환경에 솔루션이 배포되고, 면접관 에이전트에게 말을 걸면 지원자 리스트가 자동으로 세팅된다
> **예상 시간**: 10분 · **완성 신호**: SharePoint **지원자** 리스트에 지원자 42명이 채워져 있음

오전 **Lab 1~3은 50분 통합 타이머**로 진행합니다(스텝마다 끊지 않음). 이 랩 예상 **10분** — 동작 사이사이 "왜 이걸 하는지"를 듣고 넘어가세요.

---

## 준비

오늘 쓸 도구가 담긴 **솔루션 파일**(`.zip`)을 제공받습니다. 이 솔루션 하나에 **면접관 에이전트**와 **환경 세팅 흐름**이 함께 들어 있습니다. 파일을 바탕화면 등 찾기 쉬운 곳에 저장해 두세요.

![촬영: 제공받은 솔루션 zip 파일 (다운로드/바탕화면)](https://placehold.co/736x490/EEE/31343C "lab1-01")

---

## 단계

1. 브라우저에서 `https://make.powerapps.com` 으로 이동합니다. 우측 상단 **환경 선택기**를 클릭해 **교육용 환경**을 선택합니다.

    ![촬영: Power Apps 우측 상단 환경 선택기에서 교육용 환경 선택](https://placehold.co/736x490/EEE/31343C "lab1-02")

2. 왼쪽 메뉴에서 **솔루션**을 클릭합니다.

    ![촬영: 왼쪽 메뉴의 '솔루션' 항목](https://placehold.co/736x490/EEE/31343C "lab1-03")

3. 상단 명령 모음에서 **가져오기 → 솔루션 가져오기**를 클릭합니다.

    ![촬영: 상단 '가져오기' 버튼](https://placehold.co/736x490/EEE/31343C "lab1-04")

4. **찾아보기**를 클릭해 준비해 둔 `.zip` 파일을 선택하고 **다음**을 누릅니다.

    ![촬영: zip 파일 선택 후 다음 버튼](https://placehold.co/736x490/EEE/31343C "lab1-05")

5. 솔루션 정보(이름·버전)가 표시됩니다. 내용을 확인하고 **다음**을 누릅니다.

    ![촬영: 솔루션 정보 화면의 다음 버튼](https://placehold.co/736x490/EEE/31343C "lab1-06")

6. **연결** 단계가 나옵니다. **SharePoint** 연결 옆 드롭다운에서 내 계정 연결을 선택합니다. 목록에 없으면 **+ 새 연결**을 눌러 새 탭에서 로그인한 뒤, 이 화면으로 돌아와 **새로 고침** 아이콘을 누르고 다시 선택합니다.

    ![촬영: 연결 드롭다운에서 SharePoint 연결 지정](https://placehold.co/736x490/EEE/31343C "lab1-07")

    {: .note }
    연결은 "이 흐름이 어떤 계정으로 SharePoint에 접근할지"를 정하는 단계입니다. 한 번 만들어 두면 오늘 나머지 랩에서 재사용됩니다.

7. **가져오기**를 클릭합니다. 화면 상단에 진행 표시줄이 나타나며, 완료까지 잠시(1~2분) 걸립니다.

    ![촬영: '가져오기' 클릭 후 진행 표시줄](https://placehold.co/736x490/EEE/31343C "lab1-08")

8. **"솔루션을 성공적으로 가져왔습니다"** 메시지를 확인합니다. 목록에 새 솔루션이 보이면 성공입니다.

    ![촬영: 가져오기 완료 메시지 + 솔루션 목록](https://placehold.co/736x490/EEE/31343C "lab1-09")

9. 새 탭에서 `https://copilotstudio.microsoft.com` 으로 이동합니다. 우측 상단 환경이 **8단계와 같은 교육용 환경**인지 확인합니다.

    ![촬영: Copilot Studio 환경 확인](https://placehold.co/736x490/EEE/31343C "lab1-10")

10. 왼쪽 메뉴 **에이전트**를 클릭하고, 목록에서 **면접관 에이전트**를 클릭해 엽니다.

    ![촬영: 에이전트 목록의 '면접관 에이전트'](https://placehold.co/736x490/EEE/31343C "lab1-11")

11. 화면 오른쪽 **테스트** 패널이 열려 있는지 확인합니다. (닫혀 있으면 우측 상단 **테스트**를 클릭합니다.)

    ![촬영: 오른쪽 테스트 패널](https://placehold.co/736x490/EEE/31343C "lab1-12")

12. 테스트 입력창에 아래 문장을 그대로 입력하고 Enter를 누릅니다.

    ```
    환경 세팅 시작해줘
    ```

    ![촬영: 테스트 입력창에 '환경 세팅 시작해줘' 입력](https://placehold.co/736x490/EEE/31343C "lab1-13")

13. 에이전트가 **연결을 허용해 달라는 카드**를 띄웁니다. 카드의 **연결**(또는 **허용**)을 클릭하고, 안내에 따라 로그인/허용합니다. (내 계정으로 리스트를 만들기 위한 한 번의 동의입니다.)

    ![촬영: 대화창의 연결 동의 카드와 '허용' 버튼](https://placehold.co/736x490/EEE/31343C "lab1-14")

    {: .note }
    허용 후 에이전트가 멈춘 것처럼 보이면, 입력창에 `환경 세팅 시작해줘`를 한 번 더 입력하세요. 연결 직후 첫 요청이 끊기는 경우가 있습니다.

14. 에이전트가 작업을 마치고 **무엇을 만들었는지 보고**합니다. 아래와 비슷한 메시지가 나오는지 확인합니다.

    ```
    ✓ 지원자 리스트 생성 완료
    ✓ 컬럼 8개 구성
    ✓ 지원자 42명 적재
    ```

    ![촬영: 에이전트의 세팅 완료 리포트 메시지](https://placehold.co/736x490/EEE/31343C "lab1-15")

15. 새 탭에서 교육용 **SharePoint 사이트**로 이동해 **지원자** 리스트를 엽니다. 행이 42개 채워져 있는지 확인합니다.

    ![촬영: SharePoint 지원자 리스트에 42행이 보이는 화면](https://placehold.co/736x490/EEE/31343C "lab1-16")

---

## 확인

- [ ] 솔루션이 가져오기 완료되었다
- [ ] 에이전트가 세팅 리포트를 출력했다
- [ ] SharePoint **지원자** 리스트에 지원자 42명 + 컬럼 8개가 있다
